¿Que temas aprendió?

A lo largo de la presente semana, se exploraron conceptos esenciales en React, destacando la aplicación eficaz del hook useEffect para gestionar efectos secundarios. Se aprendió a estructurar la lógica de formularios mediante la creación de hooks personalizados, como el denominado useForm, y se desarrolló la capacidad de consumir APIs externas utilizando Axios. El uso de referencias (useRef) facilitó el acceso directo a nodos específicos en el DOM. Se implementaron contadores y generadores de números aleatorios para interactuar con APIs, y se fomentó la organización de componentes en archivos independientes como medida para mejorar la reutilización y la mantenibilidad del código.

Definiciones

Hook useEffect

Un hook de React que permite ejecutar efectos secundarios en componentes funcionales, como realizar solicitudes HTTP, manipular el DOM o suscribirse a eventos, en diferentes puntos del ciclo de vida del componente.

Custom Hook

Una función personalizada en React que puede contener lógica de estado y efectos, permitiendo encapsular y reutilizar comportamientos específicos en múltiples componentes.

Referencia (useRef)

Un objeto proporcionado por React que permite mantener una referencia mutable a un elemento del DOM. Se utiliza para acceder y modificar directamente propiedades o estilos del elemento sin afectar el estado de React.

Axios

Una biblioteca JavaScript utilizada para realizar solicitudes HTTP en aplicaciones basadas en el navegador y en Node.js. Es comúnmente empleada para consumir APIs externas.

Procedimientos

  1. Implementación de Efectos con useEffect en Diversos Contextos: Se llevó a cabo la aplicación de efectos con useEffect en diferentes situaciones, abarcando casos de renderización, cambios en variables particulares y ejecución única al cargar el componente.
  2. Desarrollo de un Hook Personalizado llamado useForm: SSe creó un hook personalizado denominado useForm para administrar formularios de manera más eficiente y reutilizable.
  3. Consumo de la API de Breaking Bad: Se utilizó la API de Breaking Bad para mostrar cotizaciones al recargar el componente y actualizar la interfaz mediante un botón de recarga.
  4. Creación de un Proyecto de Consumo de la API de Pokémon: Se llevó a cabo la creación de un proyecto que consume la API de Pokémon, presentando información de un Pokémon específico según un número aleatorio o seleccionado por el usuario.
  5. Uso de Axios para Solicitudes HTTP a la API de Pokémon: Se empleó Axios para realizar solicitudes HTTP a la API de Pokémon y obtener datos dinámicos.
  6. Manipulación del DOM con el Uso de Referencias (useRef): Se realizó la manipulación del DOM utilizando referencias (useRef) para cambiar estilos o interactuar con elementos específicos.

Resultados

  1. Se consiguió desarrollar un proyecto que hace uso de la API de Breaking Bad y otro que se conecta con la API de Pokémon, evidenciando la habilidad para integrar datos externos en aplicaciones React.
  2. Se introdujeron contadores y generadores de números aleatorios para incrementar la interactividad de las aplicaciones y ofrecer funcionalidades más dinámicas.
  3. La estructura del código se optimizó al dividir los componentes en archivos independientes y al aplicar prácticas de desarrollo recomendadas.
  4. Se fortalecieron los fundamentos clave de React, incluyendo el manejo de useState, useEffect, hooks personalizados y la manipulación eficaz del DOM.
