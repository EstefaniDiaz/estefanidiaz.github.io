¿Que temas aprendió?

En la presente semana, se abordó la integración de la autenticación con Next.js a través de la utilización de NextAuth. Se llevó a cabo la instalación y configuración del backend del sistema, se examinó la documentación de NextAuth referente a los proveedores de credenciales, y se procedió a la creación del frontend utilizando Bootstrap. En la configuración de NextAuth, se incluyó la creación de proveedores de credenciales destinados a la autenticación mediante correo electrónico y contraseña. Asimismo, se instaló y empleó el paquete Bootstrap con el fin de mejorar el diseño y la apariencia del frontend. Se implementó la gestión de sesiones, explorando conceptos tales como el proveedor de sesiones y el middleware de NextAuth.

Definiciones

NextAuth

Librería utilizada con Next.js para la implementación de autenticación y autorización en aplicaciones web.

Backend

Parte de una aplicación que se encarga de procesar datos y realizar operaciones del lado del servidor.

Frontend

Interfaz de usuario y experiencia de usuario de una aplicación, visible y accesible para el usuario final.

Bootstrap

Marco de diseño que facilita el desarrollo de interfaces web responsivas y atractivas.

Middleware

Software que se encuentra entre el sistema operativo y las aplicaciones, facilitando la comunicación y el procesamiento de datos.

Proveedor de sesiones

Componente que gestiona la información de la sesión del usuario, como su estado de autenticación.

Procedimientos

  1. Instalación y configuración del backend del sistema, lo cual incluyó la ejecución de comandos como npm i y docker compose up -d.
  2. Creación de un proveedor de sesiones destinado a administrar la autenticación y autorización del usuario.
  3. Configuración de proveedores de credenciales para posibilitar la autenticación mediante correo electrónico y contraseña.
  4. Instalación del paquete Bootstrap con el objetivo de mejorar el diseño y la apariencia del frontend.
  5. Implementación de un componente de botones encargado de gestionar la autenticación y desautenticación del usuario.
  6. Configuración de middleware para manejar rutas protegidas en el dashboard de la aplicación.

Resultados

  1. Se completó la instalación y configuración del backend del sistema, y se llevó a cabo la creación del frontend utilizando Bootstrap para mejorar la interfaz de usuario.
  2. NextAuth se implementó de manera exitosa para facilitar la gestión de la autenticación mediante correo electrónico y contraseña.
  3. Se diseñó y creó una página de dashboard protegida, la cual únicamente es accesible para usuarios autenticados.
  4. Se incorporaron botones de autenticación y desautenticación, permitiendo a los usuarios interactuar con el sistema de autenticación.
  5. Además, se exploraron y aplicaron conceptos importantes, tales como el proveedor de sesiones y el middleware, para gestionar el estado de autenticación y proteger rutas específicas dentro de la aplicación.
