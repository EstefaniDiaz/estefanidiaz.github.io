¿Que temas aprendió?

En esta semana, se aprendió acerca de la relevancia de la plataforma web abierta como norma para desarrollar aplicaciones en línea. Se exploró la estructura y contenido de las páginas web mediante HTML, así como su diseño y presentación con CSS. También se ofreció una introducción al diseño de páginas y su disposición utilizando técnicas como Flexbox y Grid Layout. Estos conocimientos nos brindaron una base sólida para crear y diseñar aplicaciones web.

Definiciones

La Open Web Platform, que incluye tecnologías como HTML5, CSS, SVG, ECMAScript y WAI-ARIA, desempeña un papel crucial en el desarrollo de aplicaciones y sitios web contemporáneos. Estos conceptos nos capacitan para organizar el contenido, establecer estilos y diseños atractivos, incorporar interactividad y dinamismo, y mejorar la accesibilidad para personas con discapacidades. Al dominar estas tecnologías, podemos crear experiencias web integrales y gratificantes, adaptadas a distintos dispositivos, asegurando la usabilidad, funcionalidad y estética visual de los sitios web.

Open Web Platform

Es una colección de tecnologías desarrolladas como estándares abiertos por el Consorcio World Wide Web (W3C) y otros organismos de estandarización, como el Grupo de Trabajo de Tecnología de Aplicaciones de Hipertexto Web, el Consorcio Unicode y el Internet Engineering Task Force. Esta plataforma permite a los desarrolladores crear contenido interoperable en un entorno cohesivo. Algunas de las tecnologías incluidas en la Plataforma Web Abierta son:

  • HTML (HyperText Markup Language): Lenguaje de publicación de la Web
  • CSS (Cascading Style Sheets): Lenguaje de hojas de estilo utilizado para describir la presentación de un documento HTML.
  • SVG (Scalable Vector Graphics): Formato de gráficos vectoriales basado en XML
  • Web Workers: API para la ejecución de scripts en segundo plano
  • Geolocation API: API para obtener la ubicación del usuario
  • DOM Events: API para la manipulación de eventos en el Document Object Model
  • Cross-Origin Resource Sharing: Mecanismo para permitir que los recursos de un origen sean utilizados por otro origen
Estas tecnologías, entre otras, forman la base de la Plataforma Web Abierta, permitiendo el desarrollo de aplicaciones web ricas e interoperables.

Unidades de medida en CSS

Las unidades de medida en CSS se utilizan para dimensionar elementos en una página web. Estas unidades pueden ser absolutas o relativas, y existen varias opciones para cada tipo. A continuación se muestra una lista de las unidades de medida más comunes en CSS:

  • Unidades de medida absolutas:
    • in: pulgadas, equivalentes a 2.54 cm.
    • cm: centímetros.
    • mm: milímetros.
    • q: cuarto de milímetro, 1q = 0.248 mm.
    • pt: puntos, 1pt = 1/72 de pulgada o 0.35 mm.
    • pc: paicas, 1pc = 12 puntos.
  • Unidades de medida relativas:
    • em: relativa al tamaño del elemento padre.
    • rem: relativa al tamaño base del documento HTML.
    • ex: relativa a la altura de una "x" minúscula.
    • ch: relativa al ancho de un carácter "0" (cero).
    • vw: relativa al 1% del ancho de la ventana.
    • vh: relativa al 1% de la altura de la ventana.
    • vmin: relativa al 1% del tamaño mínimo entre el ancho y la altura de la ventana.
    • vmax: relativa al 1% del tamaño máximo entre el ancho y la altura de la ventana.
  • Unidades de medida adicionales:
    • %: porcentaje, relativo al valor del elemento padre.
    • fr: fracciones, se utiliza en diseños de cuadrícula (grid layouts).
    • auto: el navegador calcula automáticamente el valor.
    • fit-content: el tamaño se ajusta al contenido del elemento.
    • min-content: el tamaño mínimo necesario para mostrar todo el contenido.
    • max-content: el tamaño máximo posible para mostrar todo el contenido.

CSS (Cascading Style Sheets)

CSS se utiliza en conjunto con HTML para crear páginas web visualmente atractivas y bien estructuradas. Los desarrolladores web utilizan CSS para definir la apariencia de los elementos HTML, como el tamaño y el color de la fuente, el color de fondo, el espaciado entre líneas, el espaciado entre elementos, etc. Al separar la presentación del contenido, CSS permite a los desarrolladores web crear páginas web más eficientes y flexibles

HTML (HyperText Markup Language)

La estructura de una página web se basa en HTML, que proporciona el esqueleto de la página. Sin embargo, HTML por sí solo no es suficiente para hacer que una página web sea visualmente atractiva e interactiva.

Procedimientos

  • Creación del Document Object Model (DOM): Se creó el DOM a partir de la respuesta HTML del servidor, esto implico convertir el código HTML en una estructura de árbol que representa la jerarquía de elementos de la página.
  • Creación del CSS Object Model (CSSOM): Se creó el CSSOM, que es una representación del CSS utilizado en la página, esto implico analizar y procesar las reglas CSS para determinar los estilos aplicados a cada elemento del DOM.
  • Combinación del DOM y el CSSOM en el Render Tree: Se combino el DOM y el CSSOM para crear el Render Tree, que es una representación de los elementos visibles en la página y sus estilos aplicados, esto implico determinar qué elementos se mostrarán en la pantalla y cómo se verán.
  • Cálculo del layout: Una vez que se tuvo el Render Tree, se calcula el layout, es decir, se determina la posición y el tamaño exactos de cada elemento en la página, esto implico establecer las dimensiones y la ubicación de los elementos en relación con otros elementos y con el contenedor principal.

Resultados

  1. Se generó el DOM a partir de la respuesta HTML del servidor, lo que posibilitó la representación de la estructura y el contenido de la página web.
  2. Se creó el CSSOM para representar los estilos CSS utilizados en la página, permitiendo la aplicación de estilos específicos a cada elemento del DOM.
  3. La combinación del DOM y el CSSOM dio lugar al Render Tree, que muestra los elementos visibles en la página junto con sus estilos aplicados. Esto determinó qué elementos se mostrarían en la pantalla y cómo se visualizarían.
