¿Que temas aprendió?

Esta semana se cubrieron varios temas relacionados con Next.js, un framework de React diseñado para aplicaciones web de pila completa. Se exploraron las Single Page Applications (SPAs) y sus limitaciones en términos de SEO, resaltando la manera en que Next.js aborda este problema. Además, se proporcionó instrucción sobre cómo emplear Server Components en Next.js para interactuar con la API de Rick and Morty, descargando proyectos desde GitHub, instalando dependencias y realizando solicitudes HTTP mediante fetch. Se introdujo el concepto de Fetch en Server Components y se explicó detalladamente la creación de funciones para obtener datos de la API y presentarlos en componentes.

Definiciones

Next.js

Un marco de React para crear aplicaciones web de pila completa, que combina la creación de interfaces de usuario mediante React Components con funciones y optimizaciones adicionales proporcionadas por Next.js.

Single Page Applications (SPAs)

Aplicaciones web que cargan una sola página y actualizan su contenido dinámicamente a medida que los usuarios interactúan con la aplicación, sin necesidad de recargar la página completa.

SEO-friendly

Característica que indica que una aplicación web o sitio es fácilmente indexable por los motores de búsqueda, permitiendo una mejor visibilidad en los resultados de búsqueda.

API REST

Interfaz de programación de aplicaciones basada en el estilo arquitectónico Representational State Transfer (REST), que utiliza métodos HTTP para realizar operaciones en recursos.

Incremental Static Regeneration (ISR)

Interfaz de programación de aplicaciones basada en el estilo arquitectónico Representational State Transfer (REST), que utiliza métodos HTTP para realizar operaciones en recursos.

Server Components

Componentes en Next.js que se ejecutan en el servidor y permiten realizar operaciones del lado del servidor, como el acceso a bases dse datos o la obtención de datos externos.

Procedimientos

  1. Descargar proyectos desde GitHub y realizar la configuración del entorno.
  2. Instalar las dependencias necesarias para el proyecto mediante el uso de comandos, como npm install.
  3. Desarrollar funciones que permitan efectuar solicitudes a una API externa utilizando la función fetch.
  4. Utilizar Server Components en Next.js para obtener y representar datos de la API de Rick and Morty.
  5. Emplear Next Image para exhibir imágenes de personajes, asegurándose de incluir propiedades obligatorias como src, width, height y alt.
  6. Realizar la implementación de la vinculación de enlaces para facilitar la navegación entre las distintas páginas de personajes.

Resultados

  1. Se logró realizar la conexión con la API de Rick and Morty mediante un Server Component en Next.js, extrayendo datos de personajes y presentándolos en la interfaz de usuario.
  2. Se llevó a cabo la implementación de la generación estática de páginas utilizando ISR, lo que contribuyó a mejorar la velocidad de carga al aprovechar los beneficios de la generación estática y actualizar las páginas de manera incremental.
  3. Se creó una página detallada para cada personaje, incorporando información como el nombre, la especie y una representación visual.
  4. Se aplicó la técnica de vinculación de enlaces para simplificar la navegación entre las diversas páginas dedicadas a los personajes.
