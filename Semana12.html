¿Que temas aprendió?

Durante el transcurso de esta semana, se adquirió conocimiento sobre el uso fundamental de useReducer y useContext como herramientas clave en el desarrollo. Se llevaron a cabo los siguientes procedimientos: se implementó useReducer para gestionar el estado de la aplicación, creando un reducer específico para controlar acciones como la adición de usuarios; la lógica de manejo de formularios se modularizó mediante la creación del hook personalizado useForm; los componentes fueron estructurados en archivos independientes para mejorar la organización y fomentar la reutilización del código; se incorporó el uso de referencias (useRef) para acceder directamente a nodos del DOM; se realizaron solicitudes a APIs externas con la ayuda de la biblioteca Axios; se implementaron alertas visuales utilizando la librería SweetAlert2. Además, se empleó useContext para transmitir eficientemente valores de estado entre componentes, separando la lógica de presentación y simplificando la estructura del proyecto. En resumen, se logró un proyecto bien estructurado, aplicando conceptos esenciales de React para desarrollar un CRUD funcional y modular.

Definiciones

useReducer

Hook de React que permite manejar el estado de la aplicación de una manera más compleja que useState, útil para casos donde el estado depende del anterior o se deben realizar acciones más complejas.

useContext

Otro Hook de React que proporciona una forma de pasar datos a través del árbol de componentes sin necesidad de pasar props manualmente en cada nivel. Reducer: Una función que especifica cómo el estado de una aplicación cambia en respuesta a acciones enviadas a través de dispatch en useReducer.

Custom Hook

Funciones personalizadas en React que encapsulan lógica específica y pueden ser reutilizadas en varios componentes.

useRef

Hook que proporciona una forma de acceder a nodos del DOM directamente en lugar de depender del estado de React, útil para realizar operaciones directas sobre elementos HTML.

Procedimientos

  1. Implementación de useReducer: Se incorporó useReducer para gestionar el estado de la aplicación, lo que posibilita transiciones de estado más complejas y el manejo de acciones específicas de manera eficiente.
  2. Creación de useForm: Se creó el hook personalizado useForm con el objetivo de modularizar la lógica asociada al manejo de formularios, simplificando así la captura y actualización de valores de los campos del formulario.
  3. Estructuración de componentes: La aplicación fue estructurada con componentes organizados en archivos independientes, una práctica que mejora la reutilización y facilita el mantenimiento del código, siguiendo las buenas prácticas de organización.
  4. Consumo de APIs con Axios: Se empleó la biblioteca Axios para realizar peticiones HTTP, permitiendo la conexión y la obtención de datos desde APIs externas de manera eficaz.
  5. Uso de referencias (useRef): Se utilizó el hook useRef para acceder directamente a nodos del DOM, eliminando la necesidad de depender del estado de React para obtener y manipular elementos específicos de manera más directa.

Resultados

  1. Mejora eficiente del estado: Mediante la incorporación de useReducer en lugar de useState, se logró gestionar el estado en la aplicación de manera más organizada y adaptable, posibilitando una transición sin problemas entre la asignación de estados y la despacho de acciones.
  2. Modularización de la lógica de formularios: La creación y aplicación del hook personalizado useForm facilitó la modularización y reutilización de la lógica asociada a los formularios. Esto simplificó la gestión de cambios en los inputs y el reinicio de los valores del formulario.
  3. Consumo eficaz de APIs externas: La utilización de la biblioteca Axios para realizar solicitudes a APIs externas proporcionó un manejo robusto de operaciones asíncronas y mejoró la integración de datos externos en la aplicación React.
  4. Acceso directo a nodos del DOM: La implementación de useRef permitió el acceso directo a los nodos del DOM, eliminando la necesidad de depender del estado de React para operaciones específicas. Esto mejoró la eficiencia en la interacción con elementos HTML sin desencadenar re-renderizaciones innecesarias.
  5. Estructuración clara de componentes:La organización de los componentes en archivos separados facilitó la reutilización y el mantenimiento del código. La división de responsabilidades en componentes específicos mejoró la legibilidad y la escalabilidad del proyecto./li>
  6. Implementación de contextos y reducers: Se empleó el uso de contextos y reducers para compartir y gestionar el estado global en la aplicación, facilitando la comunicación entre componentes y permitiendo una gestión más eficiente de las actualizaciones de estado.
