¿Que temas aprendió?

Durante la sexta semana, se profundizó en el estudio de React y su funcionalidad. React, una biblioteca de JavaScript ampliamente adoptada, se emplea para crear interfaces de usuario interactivas y escalables. Este periodo se enfocó en los principios fundamentales de React, desde la instalación inicial hasta la comprensión de las estructuras de directorio comunes en el desarrollo de aplicaciones React. Además, se exploraron los conceptos de Node.js, un entorno de ejecución esencial para trabajar con React. Node.js posibilita la ejecución de código JavaScript fuera del navegador y proporciona herramientas y bibliotecas para el desarrollo web moderno.

Definiciones

En el contexto de React, es crucial tener una comprensión sólida de ciertas definiciones clave para establecer los fundamentos de nuestro entendimiento.

React

Es una biblioteca de código abierto en JavaScript centrada en componentes, diseñada para crear interfaces de usuario dinámicas y rápidas. Desarrollada por Facebook, es una de las herramientas más usadas en JavaScript para construir aplicaciones web y móviles. React se emplea principalmente para la creación del frontend de las aplicaciones web, garantizando una renderización veloz de las páginas y un alto rendimiento.

Componente

En React, un componente representa una unidad individual y reutilizable de la interfaz de usuario utilizada para desarrollar aplicaciones web. Estos componentes permiten dividir la interfaz de usuario en partes independientes y reutilizables, facilitando la visualización y el mantenimiento del código. La modularidad de los componentes en React simplifica la creación y actualización continua de aplicaciones web, además de ofrecer una forma eficaz de gestionar la complejidad del estado de la aplicación.

Node.js

Es un entorno de ejecución de JavaScript de código abierto y multiplataforma que permite a los desarrolladores crear aplicaciones de red y del lado del servidor escalables y eficientes, con una arquitectura de programación asíncrona y sin bloqueos.

Estado

Es un objeto que contiene datos que afectan la representación visual del componente y que pueden cambiar a lo largo del tiempo, y se actualiza utilizando el método setState().

Procedimientos

  1. Configuración de Node.js: Este proceso implica la descarga e instalación de Node.js en el sistema, que se puede obtener desde el sitio web oficial de Node.js (https://nodejs.org) y está disponible para varios sistemas operativos. La instalación de Node.js proporcionará el entorno de ejecución necesario para ejecutar código JavaScript fuera del navegador.
  2. Inclusión de React y ReactDOM mediante un CDN: En lugar de descargar y agregar localmente las bibliotecas de React y ReactDOM, se ha elegido utilizar un CDN (Content Delivery Network). Este es un servicio que almacena y distribuye archivos estáticos, como las bibliotecas de JavaScript, en servidores ubicados en diversas ubicaciones geográficas para mejorar la velocidad de entrega.

.

Resultados

  1. Preparación del entorno de desarrollo: Al instalar Node.js, creamos un entorno de ejecución para JavaScript que habilita el uso de React tanto en el servidor como en el cliente.
  2. Utilización de las bibliotecas de React: Al incorporar las bibliotecas React y ReactDOM mediante un CDN, obtenemos acceso directo a las funciones y capacidades de React.
  3. Simplificación del proceso inicial: Al configurar un proyecto en React, se estableció una estructura de carpetas y se generaron los archivos de configuración necesarios, ahorrando tiempo en la configuración inicial.
  4. Desarrollo y mantenimiento sencillos: Al combinar React con Node.js y las bibliotecas React y ReactDOM, accedemos a un conjunto de herramientas y bibliotecas que simplifican tanto el desarrollo como el mantenimiento de la aplicación.
