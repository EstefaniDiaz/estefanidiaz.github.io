¿Que temas aprendió?

Se adquirieron destrezas esenciales en React, que incluyen la construcción de componentes modulares con interfaces definidas, la utilización de JSX para organizar la aplicación mediante código reutilizable, la comprensión del proceso de renderizado en React a través de ReactDOM.render(), la transferencia de datos entre componentes (tanto de padre a hijo, como de hijo a padre o entre componentes anidados), y la implementación de una estructura organizada al dividir los componentes en archivos separados para promover la reutilización y la mantenibilidad del código. Estas habilidades establecen una sólida base para el desarrollo efectivo en React.

Definiciones

JSX y React

JSX es una extensión de JavaScript utilizada en React para crear código reutilizable y estructurar la aplicación en base a componentes.

Renderizado en React

React renderiza HTML mediante ReactDOM.render().

Pasar datos entre componentes

Se refiere a la acción de enviar datos entre componentes, ya sea de padre a hijo, de hijo a padre, o entre componentes anidados.

Estructura de componentes en archivos separados

Se recomienda dividir los componentes en archivos separados para facilitar la reutilización del código.

Componente de función y de clase

En React, nos encontramos con dos tipos de componentes: los de función y los de clase. Los componentes de función producen HTML y se destacan por su sintaxis más breve, a diferencia de los componentes de clase, que requieren la declaración extends React.Component y la inclusión de un método render().

Procedimientos

  1. Desarrollo de elementos en React: Se generaron elementos en React, los cuales son unidades modulares con interfaces claramente definidas que permiten la interacción entre componentes y la capacidad de reutilizar el código.
  2. Transferencia de datos entre componentes: Se realizó la transmisión de datos entre componentes, ya sea desde el componente padre al hijo, del hijo al padre o entre componentes anidados.
  3. Organización de componentes en archivos independientes: Se fragmentaron los componentes en archivos separados para simplificar la reutilización del código.
  4. Implementación de JSX en React: Se empleó JSX, una extensión de JavaScript usada en React para crear código reutilizable y estructurar la aplicación en función de componentes.
  5. Proceso de renderizado en React: Se empleó ReactDOM.render() para presentar HTML en React. Estos procedimientos fueron llevados a cabo con el propósito de comprender y aplicar los principios fundamentales de React.

Resultados

  1. Se generaron elementos en React, siguiendo la sugerencia de descomponer el código en componentes con interfaces claramente definidas para posibilitar la comunicación entre ellos.
  2. Se llevó a cabo la transferencia de datos entre componentes, permitiendo el envío de un objeto con tres propiedades desde el componente 1 al componente 5, donde se visualizaron dichos datos.
  3. La fragmentación de los componentes en archivos independientes se realizó, facilitando así la reutilización del código y aportando una estructura organizada a la aplicación.
  4. La aplicación de JSX para organizar la aplicación basada en componentes permitió la creación de código reutilizable y la representación eficiente del HTML.
  5. El renderizado de HTML en React se logró mediante la utilización de ReactDOM.render().
